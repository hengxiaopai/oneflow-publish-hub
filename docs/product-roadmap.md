# Product Roadmap

## Phase 1：前端高保真原型

**目标**

验证编辑器主导的空间发布中枢、真实渠道状态模型与 Liquid Glass 使用边界。

**核心功能**

- 三栏写作与发布工作台。
- AI Publish Agent 阶段轨道。
- 四类发布队列、渠道选择、确认、授权、重试和预览。
- 固定发布控制条、桌面和平板响应式布局。
- Reduced Motion、Reduced Transparency 与高对比度降级。

**技术风险**

- 模拟状态被误解为真实平台能力。
- 高信息密度导致低对比度或小触控目标。
- 浏览器对 `backdrop-filter` 和透明度偏好支持不一致。

**验收标准**

- 14 个渠道均有清晰且保守的能力状态。
- 只有自建 Blog 被标为已验证自动发布。
- 桌面端编辑器占工作区约 50% 至 55%，平板使用抽屉。
- 自动测试、浏览器交互和视觉截图通过。

## Phase 2：本地草稿与发布记录持久化

**目标**

让文章、平台版本、发布任务与状态在刷新或重启后可恢复。

**核心功能**

- IndexedDB 或本地数据库保存文章与平台版本。
- 自动保存、版本历史、发布批次和失败记录。
- 导入导出 Markdown 与 JSON 任务包。
- 明确的数据迁移和清理策略。

**技术风险**

- 富文本、Markdown 与平台版本之间的数据一致性。
- 本地存储配额、损坏恢复和并发编辑冲突。
- 敏感授权信息不能写入普通前端存储。

**验收标准**

- 刷新后文章、选择项和发布记录可恢复。
- 数据升级有版本号和迁移测试。
- 清除数据前有明确确认，损坏记录可回退。

## Phase 3：自建 Blog / Halo 发布打通

**目标**

完成第一个真实、可验证、可回滚的自动发布渠道。

**核心功能**

- 自建 Blog Webhook 或 REST API 发布器。
- Halo API 认证、文章创建、更新、分类、标签和封面上传。
- 幂等键、防重复发布、状态轮询和链接回写。
- 草稿发布、正式发布和撤回策略。

**技术风险**

- Halo 版本差异、认证方式和媒体上传限制。
- Markdown、HTML 与编辑器格式转换。
- 网络失败造成重复文章或部分成功。

**验收标准**

- 测试站点可创建并更新文章。
- 同一发布任务重复执行不会产生重复内容。
- 失败有可理解的错误、重试和人工恢复路径。
- 发布结果链接与时间可回写。

## Phase 4：第三方平台半自动适配与复制发布

**目标**

在不承诺全自动的前提下，提高第三方文章平台的发布效率。

**核心功能**

- 每个平台的标题、摘要、标签、封面和正文转换器。
- 本地平台草稿与复制发布包。
- 一键复制、打开目标平台、检查清单和人工确认。
- 平台规则版本与能力验证记录。

**技术风险**

- 平台编辑器规则变化频繁。
- 图片转存、代码块、链接和 Markdown 兼容性差异。
- 剪贴板权限和登录态不可控。

**验收标准**

- 每个平台版本可独立预览和编辑。
- 复制内容不丢失关键结构。
- UI 明确区分已生成、已复制、已人工发布和待验证。

## Phase 5：浏览器自动化或 API 发布器实验

**目标**

对具备条件的平台实验 API 或浏览器自动化发布，不把实验能力默认视为稳定功能。

**核心功能**

- OAuth、权限申请和账号连接管理。
- 平台发布器插件协议。
- 浏览器自动化实验、选择器版本管理和可观察日志。
- 用户在最终发布前明确确认。

**技术风险**

- 平台条款、反自动化策略、验证码和账号安全。
- DOM 变化导致自动化失效。
- OAuth 权限回收、接口限流、审核和内容合规。
- 任何代用户发布操作都必须可感知、可取消、可追踪。

**验收标准**

- 每个平台独立开关，默认关闭实验能力。
- 自动化失败不会丢失本地草稿。
- 权限、操作日志和最终确认完整。
- 合规审查通过后才允许进入稳定渠道。

## Phase 6：数据回流与复盘分析

**目标**

将发布结果、链接和可获得的表现数据回写到文章任务，支持渠道复盘。

**核心功能**

- 发布链接、状态、审核结果与失败原因回写。
- 在授权范围内获取阅读、播放和互动数据。
- 统一指标口径、数据更新时间和来源标记。
- 生成渠道差异复盘和下一轮适配建议。

**技术风险**

- 不同平台指标定义、刷新频率与权限范围不一致。
- 数据缺失、延迟和接口限流。
- 不能用不可比指标制造虚假排名或无意义图表。

**验收标准**

- 每个指标都有来源、更新时间和口径说明。
- 无权限或无数据时显示明确空状态。
- 复盘建议可追溯到真实发布版本和数据。
- 不展示无法解释或不可比较的聚合图表。
